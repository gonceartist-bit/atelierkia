# بانک محتوای آموزشگاه

اپ مدیریت محتوای هنری برای آموزشگاه‌های نقاشی کودک/نوجوان: ثبت آثار، تولید کپشن/هشتگ/ایده‌ی ریلز، مدیریت نمایشگاه‌ها و هنرجویان.

## نکته‌ی مهم اگر قبلاً نسخه‌ی قدیمی رو نصب کرده بودی
یک باگ در فایل `sw.js` (سرویس‌ورکر، مسئول بازشدن آفلاین) پیدا و رفع شد که باعث می‌شد اپ **بار اول درست باز بشه ولی بار دوم به بعد `ERR_FAILED` بده**. نسخه‌ی جدید این مشکل رو نداره، ولی چون سرویس‌ورکر خراب قبلی ممکنه هنوز روی گوشی‌ات فعال باشه، بعد از آپلود فایل‌های جدید **یک‌بار دیگه** این کار رو بکن تا قفل بشکنه:
1. Chrome → Settings → Site settings → All sites → پیدا کردن `xxx.workers.dev` مربوط به این پروژه
2. **Clear & reset**
3. اپ رو دوباره باز کن (این‌بار نسخه‌ی درست سرویس‌ورکر نصب می‌شه و مشکل برنمی‌گرده)

## فایل‌های پروژه
```
index.html         اپ اصلی (تمام کد در یک فایل)
manifest.json       برای نصب اپ روی گوشی (PWA)
sw.js               سرویس‌ورکر برای بازشدن آفلاین
icon-192.png        آیکون اپ
icon-512.png        آیکون اپ
wrangler.jsonc      تنظیمات دیپلوی روی Cloudflare (سایت را به‌عنوان استاتیک معرفی می‌کند)
```
همه‌ی فایل‌ها باید کنار هم و در ریشه‌ی ریپازیتوری باشند (نه داخل زیرپوشه).

## گام ۱: آپلود روی گیت‌هاب
1. یک ریپازیتوری جدید در گیت‌هاب بساز (مثلاً `atelier-app`) — یا از همان ریپوی فعلی‌ات (`atelierkia`) استفاده کن.
2. تمام فایل‌ها را مستقیم در ریشه‌ی ریپازیتوری آپلود کن — بدون پوشه‌ی جدا:
   - `index.html`
   - `manifest.json`
   - `sw.js`
   - `icon-192.png`
   - `icon-512.png`
   - `wrangler.jsonc` ← فایل جدید، حتماً این را هم آپلود کن

## گام ۲: دیپلوی روی Cloudflare

کلودفلر امسال Workers و Pages را در یک رابط واحد ادغام کرده و دیگر همیشه یک تب جدا برای «فقط Pages» وجود ندارد. به همین دلیل، به‌جای تلاش برای پیدا کردن تب Pages، از روش رسمی جدید (Workers با Static Assets) استفاده می‌کنیم — که همان چیزی است که خود کلودفلر الان برای پروژه‌های جدید توصیه می‌کند.

فایل `wrangler.jsonc` که اضافه کردم دقیقاً همین کار را می‌کند: به کلودفلر می‌گوید «این فقط یک سایت استاتیک است، فقط فایل‌ها را سرو کن» — بدون نیاز به نوشتن هیچ کد سمت سروری.

### اگر پروژه‌ی `atelierkia` را قبلاً ساختی (همونی که الان `ERR_FAILED` می‌داد)
1. فایل `wrangler.jsonc` را به همان ریپوی گیت‌هاب اضافه کن (طبق گام ۱)
2. برو به داشبورد کلودفلر → پروژه‌ی `atelierkia` → منتظر بمان تا دیپلوی خودکار جدید (به‌خاطر push جدید) اجرا شود، یا دستی از تب **Deployments** گزینه‌ی **Retry deployment** / **Create deployment** را بزن
3. بعد از دیپلوی، آدرس `https://atelierkia.gonce-artist.workers.dev/` (بدون `/index.html`) را دوباره امتحان کن

### اگر می‌خواهی از صفر بسازی
1. **dash.cloudflare.com → Workers & Pages → Create application**
2. هرچی که به تو نشان داده شد (Import a repository / Connect to Git) را انتخاب کن — چون فایل `wrangler.jsonc` در ریپو هست، کلودفلر خودش تشخیص می‌دهد که این یک سایت استاتیک است و نیازی به تنظیم دستی Build command / Output directory نیست
3. ریپازیتوری را انتخاب و دیپلوی کن

اگر جایی در مسیر همچنان فیلدهای «Build command» و «Build output directory» را دیدی، همان مقادیر قبلی را بگذار: **Build command خالی، Output directory برابر `/`**.

اگر ریپو private است، باید اول به Cloudflare دسترسی‌اش را بدهی: GitHub → Settings → Applications → Installed GitHub Apps → Cloudflare Pages/Workers → Configure → ریپو را در Repository access اضافه کن.

## گام ۲: نصب روی موبایل

### اندروید (کروم)
1. آدرس گیت‌هاب Pages را در کروم باز کن.
2. یا منتظر بمان تا نوار «نصب اپ» پایین صفحه ظاهر شود، یا از داخل اپ به **تنظیمات → نصب اپ** برو و دکمه را بزن.
3. اگر آن ظاهر نشد: منوی سه‌نقطه‌ی کروم → «Add to Home screen» / «نصب برنامه».

### آیفون (سافاری)
1. آدرس را در سافاری باز کن (فقط سافاری این قابلیت را دارد، نه کروم آیفون).
2. دکمه‌ی اشتراک‌گذاری (مربع با فلش رو به بالا) را بزن.
3. «Add to Home Screen» را انتخاب کن.

بعد از نصب، اپ مثل یک اپ واقعی با آیکون روی صفحه‌ی اصلی گوشی باز می‌شود (بدون نوار آدرس مرورگر).

## نکات مهم درباره‌ی داده‌ها

- **همه‌ی اطلاعات (آثار، عکس‌ها، تنظیمات) فقط داخل همین مرورگر/گوشی ذخیره می‌شود** (با IndexedDB). اگر مرورگر را عوض کنی، Cache/داده‌های مرورگر را پاک کنی، یا گوشی را عوض کنی، این اطلاعات از بین می‌رود.
- به همین دلیل بخش **تنظیمات → پشتیبان‌گیری** اضافه شده: هر چند وقت یک‌بار «دانلود فایل پشتیبان» را بزن تا یک فایل JSON از کل داده‌ها بگیری، و در صورت نیاز از همان‌جا «بازیابی از فایل پشتیبان» را انجام بده.
- **انتخاب پوشه هنگام دانلود:** در مرورگرهایی که از آن پشتیبانی می‌کنند (مثلاً کروم/edge دسکتاپ، و بعضی نسخه‌های جدید کروم اندروید)، هنگام گرفتن فایل پشتیبان یک پنجره‌ی «ذخیره به‌عنوان» باز می‌شود تا خودت پوشه را انتخاب کنی — تا با بقیه‌ی دانلودها قاطی نشود. در مرورگرهایی که این قابلیت را ندارند (سافاری، فایرفاکس، بیشتر مرورگرهای موبایل)، فایل به‌صورت خودکار در پوشه‌ی پیش‌فرض Downloads ذخیره می‌شود؛ در این حالت بهتر است بعداً خودت آن را به پوشه‌ی دلخواه منتقل کنی.
- چاپ لیبل (`window.print`) هم از پنجره‌ی چاپ خود سیستم‌عامل/مرورگر استفاده می‌کند که معمولاً گزینه‌ی «Save as PDF» و انتخاب پوشه را از قبل دارد.
- فایل پشتیبان شامل عکس‌هاست (به‌صورت base64) بنابراین ممکن است حجمش چند مگابایت باشد؛ آن را جایی امن (ایمیل خودت، گوگل‌درایو) نگه دار.

## درباره‌ی تولید محتوا با هوش مصنوعی

اپ به این ترتیب تلاش می‌کند محتوا بسازد:
1. **Claude** — فقط وقتی این فایل داخل خود Claude (claude.ai) باز باشد کار می‌کند؛ در حالت نصب‌شده روی گوشی یا از گیت‌هاب Pages این مرحله معمولاً با خطا مواجه می‌شود و اپ خودکار به مرحله‌ی بعد می‌رود.
2. **Gemini** — اگر در تنظیمات یک کلید رایگان از Google AI Studio (aistudio.google.com) وارد کرده باشی، از این مسیر استفاده می‌شود. این تنها راه واقعی برای هوش مصنوعی واقعی در نسخه‌ی نصب‌شده روی گوشی است.
3. **قالب آفلاین** — اگر هیچ‌کدام در دسترس نبود (یا اینترنت قطع باشد)، از یک قالب متنی از پیش نوشته‌شده استفاده می‌شود. منبع تولید همیشه بالای صفحه نمایش داده می‌شود تا شفاف باشد.

## محدودیت‌های شناخته‌شده (برای توسعه‌ی بعدی)

- **ویدیو هنوز پشتیبانی نمی‌شود** — فقط عکس. برای ذخیره‌ی ویدیوهای باکیفیت باید یک سرویس ابری (Firebase Storage یا Supabase Storage) اضافه شود، چون IndexedDB برای ویدیوهای حجیم مناسب نیست و بین گوشی‌ها هم‌گام نمی‌شود.
- داده‌ها بین دستگاه‌های مختلف همگام‌سازی (Sync) نمی‌شوند؛ هرکسی که این PWA را نصب کند، بانک اطلاعاتی جدا و مستقل روی گوشی خودش دارد. اگر چند نفر (مثلاً چند مربی) باید به یک بانک محتوای مشترک دسترسی داشته باشند، نیاز به یک بک‌اند واقعی (Firebase/Supabase) خواهد بود.
- دوربین داخل اپ نیاز به اجازه‌ی دسترسی دوربین دارد و فقط روی HTTPS کار می‌کند (گیت‌هاب Pages این شرط را دارد).

## تست محلی قبل از آپلود (اختیاری)
```bash
cd atelier
python3 -m http.server 8000
# سپس در مرورگر: http://localhost:8000
```
(دقت کن: روی `localhost` هم دوربین و سرویس‌ورکر کار می‌کنند، نیازی به HTTPS محلی نیست.)
